# 16 UI パターン横断調査

**作成日**: 2026-05-19
**目的**: 筋肉塾の主要機能ごとに、6社のUIパターンを横断比較し、「いいとこどり」候補を整理する
**対象機能**: できた！ / 試験 / 目標管理シート / 進捗可視化 / バッジ・レベル / ストリーク / コメント

---

## 1. 「できた！」ボタン / 達成感UIパターン

筋肉塾の **核機能 M-2**。「小さな成功体験」を視覚的に祝う UI のバリエーション。

| サービス | パターン | 演出 |
|---|---|---|
| Duolingo | レッスン完了画面に紙吹雪+キャラ歓喜+XP獲得数表示 | 派手、感情を動かす |
| Trailhead | バッジ獲得+紙吹雪 | バッジが視覚的記憶に残る |
| Kajabi | チェックマーク + 自動次レッスンへ進む | 機能的、淡白 |
| Skool | レッスンに「Completed」表記、ポイント獲得 | シンプル |
| Schoo | 学習履歴に追加されるだけ | 控えめ |
| SHElikes | コース完了で次のコース推奨 | フィードバック弱め |

### 筋肉塾 推奨パターン
```
[できた！ボタン押下時]
1. ボタンが緑→ゴールドに色変化 + ぐっと拡大アニメ（0.3秒）
2. 紙吹雪エフェクト（Trailhead 流、控えめに）
3. +XP X pt 獲得 を浮かせる
4. 「累計 ◯ できた！」がカウントアップ
5. 連続記録に1日追加（既に今日カウント済みなら何もしない）
```

### バリエーション
- 章末の「できた！」= 通常版より少し派手（バッジ獲得演出）
- 試験合格時 = 最大級の祝福（紙吹雪+Lv↑表示）
- 卒業時 = 全画面演出（SNS シェアボタン併設）

## 2. 試験 / クイズ UI パターン

筋肉塾の **M-8 試験機能**。「何度でも受験可、合否+正答率」が必須要件。

| サービス | 構造 | 特徴 |
|---|---|---|
| Trailhead | Module 末尾の Quiz、不合格→該当章へ誘導 | 「ここを見直そう」リンク親切 |
| Kajabi | Assessment 機能、自動採点、合格点設定可 | 標準的 |
| Duolingo | 1問1画面、即時フィードバック | 短く集中 |
| Schoo | 一部授業内クイズ、軽め | 学習中心、試験は副次 |

### 筋肉塾 推奨パターン
```
[試験画面]
- 問題数表示（◯/全問）
- 進捗バー
- 1問1画面（or スクロール式、どちらか選択）
- 多肢選択（4択基本）
- 「次へ」ボタンで進む
- 全問解答後、結果画面へ

[結果画面]
- 正答率 ◯%
- 合否（合格点 80%）
- 合格 → 紙吹雪 + バッジ獲得演出
- 不合格 → 「該当章のレッスンを復習しよう」リンク付き
- 「再受験する」「結果を見る」「コンテンツへ戻る」ボタン
```

### 既に [database_design_draft.md](../00_premises/database_design_draft.md) と整合
- `tests` テーブル: `passing_score` デフォルト80
- `test_attempts` テーブル: `score`, `passed`, `answers`
- UNIQUE 制約なし（何度でも受験可）

## 6. ストリーク（連続記録）UI パターン

Duolingo の最強武器を **筋肉塾用に翻訳**。

### コア要件
- 全画面で**常時可視化**（小さな炎アイコン + 日数）
- **「1日1アクション」で維持**（できた！1個 / コメント1件 / 視聴1分のいずれか）
- **Streak Freeze（救済）**: 月2回まで自動消費

### 視覚デザイン
```
🔥 8日連続 → 上部に常時固定
```
- 0日 → グレー炎
- 1〜6日 → 黄色炎 + 数字
- 7日〜 → オレンジ炎 + 数字
- 30日〜 → 赤炎 + 数字
- 100日〜 → 青炎 + 数字（レア）

### 心理設計
- **損失回避**: 「ここまで続けた記録を失いたくない」
- **寛容な救済**: Streak Freeze で「忘れ」をリカバー
- **キャラ性**: のり氏キャラ風通知「待ってますよ」

## 7. コメント / コミュニティ UI パターン

筋肉塾の **M-7 コメント機能**（公開・スレッド・画像添付）。

### パターン比較

| サービス | パターン |
|---|---|
| Schoo | 動画下にコメントがリアルタイム流れる |
| Skool | Reddit 風スレッド型、独立タブ |
| Kajabi | レッスン下にディスカッション、グループ機能 |
| SHElikes | 別Slack/Discord 風コミュニティ |

### 筋肉塾 推奨パターン
**「Kajabi 風 = レッスン下にコメント」がベース**

```
[/contents/lessons/{id} レッスン詳細画面]

┌──────────────────────────┐
│   動画プレイヤー         │
└──────────────────────────┘

[できた！] [次へ >]

━━━ コメント (12) ━━━━━━━━━━━━

📌 のり氏（ピン留め）
   この章の理解ポイントは...
   👍 24

👤 山田さん 2時間前
   質問: ◯◯について...
   👍 5  💬 3
   └ 👤 鈴木さん: 私も同じ疑問でした！
   └ 📌 のり氏: ◯◯ですね。

👤 田中さん 5時間前
   できた！の自分の様子写真↓
   [写真]
   👍 12  💬 2

[+ コメントする]
[+ 画像を添付]
```

### 重要設計
- **のり氏コメントは📌で固定表示**（ピン機能）
- **画像添付可**（モバイル撮影即アップロード）
- **自分の投稿は削除可、他者投稿は閲覧のみ**
- **リアクション = 「👍」のみシンプルに**

## 8. オンボーディング UI パターン

筋肉塾の **M-11 オンボーディング**（資料3点 + 10日返金案内 + 目標シート記入）。

### Stu McLaren 流オンボーディング
- ウェルカム動画（運営者自身）
- 「ここから始めよう」のガイド
- 初回タスクのチェックリスト

### Duolingo 流オンボーディング
- 興味質問（モチベ確認）
- スキルレベル確認
- Daily Goal 設定
- 最初の小さな成功（短いレッスン1個）

### 筋肉塾 推奨フロー
```
[Step 1] ウェルカム動画（のり氏から1分）
   ↓
[Step 2] サポート期間 180日のご案内
   ↓
[Step 3] 学習の進め方（資料3点セット）
   ↓
[Step 4] 質問の仕方
   ↓
[Step 5] LINE 連携のご案内（オープンチャット参加コード）
   ↓
[Step 6] 🎯 目標管理シートの初回記入（必須）
   ↓
[Step 7] 「最初のレッスンを見る」ボタン → 章1のLesson 1 へ
```

### 設計のコツ
- **Step ナビゲーション**（◯/7）で進捗可視化
- **戻る可**（前の Step に戻れる）
- **スキップ不可は目標シートだけ**（他は後でも記入可）
- 完了時に「ようこそ！」演出 + 最初のバッジ獲得
