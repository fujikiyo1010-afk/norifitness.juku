🗺 全体構成

  • ヘッダー: ← 戻る + 「月次添削 履歴」 (シンプル)
  • ブロック A: 最新月の 4 状態カード (A 未記入 / B 記入中 / C 提出済み・返信待ち / D 返信届いた) ← 別チャット指示反映
  • ブロック B: 推移ビジュアライズ
    • B-1: カテゴリ別スコア = レーダー (全体バランス) + バー (数値詳細) を 1 ブロックに統合
    • B-2: 月次推移グラフ (折れ線)
  • ブロック C: 月別ログ一覧 (タップで詳細遷移)

※ ブロック A は 4 状態を比較できるよう縦並びで全部見せています。実装時は受講生の現在状態に応じて 1 つだけ表示。

📋 ブロック A: 4 状態カードの判定

状態条件
A 未記入monthly_audits レコードなし、または items_filled_count = 0
B 記入中submitted_at IS NULL かつ items_filled_count > 0
C 提出済みsubmitted_at IS NOT NULL かつ nori_video_published_at IS NULL
D 返信届nori_video_published_at IS NOT NULL

重要原則: 「提出ボタンを押した瞬間が、初めて のりfitness に通知される境界線」。記入中 (状態 B) は受講生個人の作業領域、のりfitness 側からは一切見えない。

⚖️ レーダー + バー 併用の役割分担

レーダー (上): 全体バランスを直感的に把握。月の傾向を一目で。
バー (下): 個別カテゴリの正確な数値 + 前回値との比較 (縦線 |)。

→ 「ざっくり概観 → 詳細数値」 の流れで自然に読める。同じデータの 2 つの見方として並列。

🎨 デザイン採用ルール

  • ブロック A の背景 = E案グラデ + 薄黄縁取り (のりfitnessコメント = 黄色 のブランド一貫)
  • 動画返信カード = 黄マーカー背景 + 名前色 #b8860b
  • カテゴリ別スコア = ティール緑 #00897b 統一
  • 折れ線グラフ = 最新月のポイントだけ大きめ + 白縁
  • 月別ログ = シンプル list 行 (overview と同じノリ)

⚠️ 現状の前提・未確定

  • 動画サムネ画像 = 黒プレースホルダー (実装時は実動画から自動生成)
  • のりfitness返信テキスト = 仮文言
  • 3 ヶ月前の比較値 = 仮ダミー値
  • レーダー軸の最小値 = 0 (案 a: 最小 5 にして凹みを緩和 / 未採用)
  • ブロック C をタップした先の 月詳細画面 = 別ファイル (monthly_review_detail.html) で並列提示
