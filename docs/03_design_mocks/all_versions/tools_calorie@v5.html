🗺 画面の役割

  • ツール一覧 → 「② 必要カロリー計算」タップ → このページ
  • ハリスベネディクト式 (改訂版) で 4 パターンのカロリーを算出
  • 「反映」押下 → 選択ダイアログ → 目標シート ③ 栄養設計「目標カロリー」に転記

📐 構成 (上から)

  • ヒーロー: ツール番号 + タイトル + 「✓ 前回 ◯月◯日 計算」表示 (DB保存方式)
  • 入力カード: 性別 (3択) + 身長 + 体重 + 年齢 + 運動量レベル (5段階)
  • 計算ボタン: インディゴ + 3px シャープ
  • 結果カード: 2×2 グリッド (基礎代謝 / メンテ / ダイエット / 増量)
    • メンテ = インディゴ系背景
    • ダイエット = ティール緑系背景
    • 増量 = オレンジ系背景
  • 計算式: 透明性のため公式を明記
  • 反映ボタン: ティール緑 + 3px (目標シート連携)

🎯 反映時 = 案 γ ハイブリッド (確定)

「反映」押下 → 選択ダイアログ表示 (4 カードからどれを反映するか)
★ 推奨マーク = 目標シート ② 目標体重 vs 現在体重 で自動判定:
・目標体重 < 現在体重 → ダイエット時 推奨
・目標体重 = 現在体重 → メンテナンス 推奨
・目標体重 > 現在体重 → 増量時 推奨
→ 初心者は推奨を選ぶだけで OK、上級者は他の値に変更可能

🧮 計算ロジック

  • 男性 BMR: 88.362 + (13.397 × W) + (4.799 × H) − (5.677 × A)
  • 女性 BMR: 447.593 + (9.247 × W) + (3.098 × H) − (4.330 × A)
  • メンテナンス = BMR × 運動量係数 (1.2 / 1.375 / 1.55 / 1.725 / 1.9)
  • ダイエット = メンテ − 500 ・ 増量 = メンテ + 500

💾 DB 保存 (案 3 確定)

  • tool_calculations テーブルに tool_id='calorie' で最新値を上書き保存
  • 次回開いた時、前回入力値がフォームに自動復元 + 結果も表示
  • ヒーロー帯に「✓ 前回 ◯月◯日 計算」表示

⚠️ 未確定 / モック上の仮置き

  • 反映ダイアログはモック上は併記表示、実装時はモーダル (背景半透明 + 中央ダイアログ)
  • 選択ダイアログのキャンセル時挙動 = ツール画面に留まる (確定)
  • 「★ 推奨」マークの判定ロジックは目標シート ② の値が入力されている前提
  • 目標シート未入力時の挙動 = 推奨マークなし、ユーザーが完全自由選択
