**作成**: 2026-06-22
**対象**: 筋肉塾アプリ 線① 完成後 ソフトローンチ (= 社内 3 人 β テスト)

## 🎯 ソフトローンチとは

```
α (= 自分) → β (= 信頼できる少数) → 本ローンチ (= 全員)
        ↑              ↑
      きよむさん     社員 3 人
      リハ済       これから
```

**1-2 週間** で 社員 3 人 (= のり氏 + 他 2 名 + きよむさん自身) に **実運用テスト** してもらう。

---

## 📝 渡し方 ・ 3 ステップ

### Step 1 ・ 招待発行 (= 1 人あたり 2 分)

1. Mac admin で [https://juku.norifitness.com/admin/invitations](https://juku.norifitness.com/admin/invitations)
2. 「新規招待」 セクションで **氏名 + 本人メアド** を入力
3. 「招待メールを送信」 押下 → 自動で **招待リンク付きメール** が送信される

### Step 2 ・ LINE / Slack で 別途案内

招待メールだけだと「迷惑メール扱いされる / 気付かない」 ことがあるので、 別チャネル (= LINE / Slack 等) で別途連絡:

**送るメッセージ案** (= コピペで OK):

```
お疲れさまです。

筋肉塾 アプリの β テスト ご協力 ありがとうございます。

📩 招待メール送りました
件名「【筋肉塾】受講開始のご案内 — パスワード設定をお願いします」

▼ お願い (= 30 分程度)
1. iPhone Safari でメール内リンクを開く
2. パスワード設定 (= 8 文字以上)
3. オンボ 8 ステップ進める (= 最後にプロテイン発送先入力あり)
4. ホーム画面追加 → アプリで再起動
5. 触れる範囲で 各機能を試してみる

▼ 気付いたこと は LINE で気軽に教えてください
- 「ここ分かりにくい」
- 「ここ壊れてる」
- 「ここ いい感じ」

何でも歓迎です。 「動かなかった」 系は スクショ + どの画面 で送ってもらえると最高!

よろしくお願いします!
```

### Step 3 ・ 必要な準備物 (= 受講生側)

- **iPhone** (= iOS 16.4 以降推奨)
- **Safari** (= デフォルト) または **Gmail** (= Safari でリンク開く設定推奨)
- **30 分** の時間 (= 最初の動線体験用)

## 🚨 きよむさんが テスト中に **気をつけること**

### A. 心構え

| 心構え | 理由 |
|---|---|
| 「完璧じゃない」 を最初に伝える | 受講生が「壊れた!」 とパニックにならない |
| バグ報告は「ありがとう」 で受ける | 「すいません」 より「教えてくれてありがとう、 直します」 |
| 即対応の体制 | LINE / Slack に常時通知 / 30 分以内に返信 |
| 期待値調整 | 「βテストなので不具合あるかも、 でも体感を一緒に作って下さい」 |

### B. 実務 ・ バグ報告管理

**スプレッドシート** で管理推奨:

| 日時 | 報告者 | 内容 | 重要度 | 対応 | 修正済? |
|---|---|---|---|---|---|
| 06/22 14:00 | のり氏 | ホームの「カルテ未記入」 が消えない | 高 | Claude に依頼 | ⏳ |
| 06/22 15:30 | 山田氏 | 月次添削の動画が再生されない | 高 | Vimeo 確認 | ✅ |
| 06/22 16:00 | 鈴木氏 | ボタンの文言が変 | 低 | 線② で修正 | - |

### C. バグ修正 → デプロイ → 受講生通知 のフロー

```
1. 受講生からバグ報告 (= LINE)
2. きよむさん → Claude に共有
3. Claude が修正 → commit + push
4. Vercel が自動デプロイ (= 3-5 分)
5. きよむさん → 受講生に「直しました、 PWA を一度閉じて開き直してください」 と LINE
6. 受講生 確認 → OK 報告
```

### D. 重大バグ時の判断

| バグの重さ | 対応 |
|---|---|
| **画面崩れ / 文言ミス** | 即修正 / 通常デプロイ |
| **入会できない / カルテ提出できない** | 緊急修正 / 受講生に「ちょっと待ってください」 連絡 |
| **データ消える / 他人のデータ見える** | 巻き戻し検討 / Claude と緊急相談 |

## 📅 期間と進捗

### 1 週目 (= 入会動線 + 基本機能)
- 月: 招待発行 + 入会動線 体験
- 火-水: カルテ / 目標シート / 体組成 提出
- 木-金: メニュー配布 + 動画再生 + チャット
- フィードバック収集 → 修正

### 2 週目 (= 継続使用 + 微調整)
- リマインド系の挙動確認 (= 数日空けて通知届くか)
- 月次添削 提出 → 動画 upload → 受信
- 細かい UX 改善 (= フィードバック反映)

### 完了判定
- **重大バグなし**
- **3 人とも「実用に耐える」 と評価**

→ **本ローンチ (= LP 公開 / 募集開始)** へ。

## 🌟 きよむさん自身が「素人」 として大事なこと

1. **「分からないこと」 を即聞く** (= Claude に聞く / 受講生に聞く / のり氏に聞く)
2. **「完璧」 を目指さない** (= ソフトローンチ = 完璧でない前提)
3. **「ありがとう」 を多く言う** (= 受講生 / Claude / 自分自身に)
4. **記録を残す** (= スプシ / メモ / Slack 等で全フィードバック保存)
5. **後で振り返れる状態にする** (= 「本ローンチで何を改善したか」)
