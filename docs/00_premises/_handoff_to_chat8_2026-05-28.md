## 🛠 Step 10b 体脂肪率計算 ・ 完全仕様

### ファイル構成

```
src/lib/tools/
  ├── types.ts         ← ToolId, BodyFatInputs, BodyFatOutputs 型定義
  ├── calculations.ts  ← 計算式 (純粋関数、Server/Client 両方で使える)
  ├── queries.ts       ← getMyToolCalculation (tool_calculations 取得)
  └── actions.ts       ← saveToolCalculation (Server Action UPSERT)

src/app/tools/body-fat/
  ├── page.tsx                    ← Server Component、前回値取得 → Client へ渡す
  └── BodyFatToolClient.tsx       ← Client Component、メイン UI
```

### 計算式 (アメリカ海軍式)

```
男性式: BF% = 495 / (1.0324 − 0.19077 × log10(W−N) + 0.15456 × log10(H)) − 450
女性式: BF% = 495 / (1.29579 − 0.35004 × log10(W+Hip−N) + 0.22100 × log10(H)) − 450
体脂肪量 (kg) = 体重 × BF% / 100
除脂肪量 (kg) = 体重 − 体脂肪量
```

W = ウエスト (cm)、N = 首回り (cm)、H = 身長 (cm)、Hip = ヒップ (女性式のみ)
※ 0〜60% の範囲にクランプ + 小数 1 桁

### 入力フォーム

- 性別 3 択 (男性/女性/その他)
- 「その他」選択時のみ formula 選択 (男性式/女性式) ボタンが追加表示
- 「女性」or「その他 → 女性式」時のみ **ヒップ欄が動的表示**
- 必須項目: 身長/ウエスト/首回り (+ 女性式時はヒップ)
- 任意: 体重 (体脂肪量/除脂肪量を出すため)

### UI 細部 (きよむさんフィードバック反映済)

1. **前回値プリセット (薄色 → 編集で通常色)**:
   - `touched` state で「触ったかどうか」を管理
   - 触ってない = `text-zinc-400` (薄色)
   - 触った = `text-zinc-900` (通常色)
   - そのまま計算ボタン押せば前回値で再計算可能

2. **未入力赤枠**:
   - `missingFields: Set<string>` state
   - 計算ボタン押下時、未入力フィールドを記録
   - InputRow に `hasError` prop で **赤い border + 薄ピンク背景** + 単位の文字も赤
   - 入力すると該当 field の `missingFields` をクリア → 通常枠に

3. **placeholder の例の数値 → 完全削除**:
   - 「バックスペースで消したら例の数値が出るのが不要」とのフィードバック
   - InputRow の `placeholder` prop 自体を削除

4. **文字色の濃さ調整 (きよむさんフィードバック反映)**:
   - 計算式の補足: `text-zinc-500` → `text-zinc-700` (フォントも 10 → 11px)
   - サブテキスト: `text-zinc-400` → `text-zinc-600`
   - 保存ボタン下の注記: `text-zinc-400` → `text-zinc-600`、サイズ 10 → 11px

### Server Action (saveToolCalculation)

- `tool_calculations` テーブルに UPSERT
- UNIQUE 制約: `(user_id, tool_id)` で同じユーザー × 同じツール = 1 件
- 保存時に `revalidatePath` で当該ツール画面を再生成 (前回値表示を更新)

### 結果カード

- メイン: 体脂肪率 (%) を 42px 大文字 + インディゴカラー
- サブ: 体脂肪量 (kg) / 除脂肪量 (kg) - 体重を入力した場合のみ表示
- 計算式の補足表示 (透明性のため)

### 保存ボタン下の注記

```
保存すると次回開いた時に前回値が復元されます
(目標シートには反映されません ・ 反映は目標シート編集画面から)
```

→ 単独モード = 目標シート不連携 を明示

## 📚 Day 10 ・ 残作業の計画

### Step 10c: 必要カロリー計算 (`/tools/calorie`)

**モック**: `/tmp/tools_calorie.html`
**計算式**: ハリスベネディクト式 (改訂版)

```
男性 BMR = 88.362 + (13.397 × W) + (4.799 × H) − (5.677 × A)
女性 BMR = 447.593 + (9.247 × W) + (3.098 × H) − (4.330 × A)
メンテナンス = BMR × 運動量係数
ダイエット = メンテ − 500
増量 = メンテ + 500
```

**入力**:
- 性別 (3 択)
- 身長、体重、年齢
- 運動量レベル (5 段階):
  - ほとんど運動しない (1.2)
  - 軽い運動 ・ 週 1〜3 回 (1.375)
  - 中程度の運動 ・ 週 3〜5 回 (1.55)
  - ハードな運動 ・ 週 6〜7 回 (1.725)
  - 非常にハード ・ 毎日 2 回など (1.9)

**出力**:
- 4 セルグリッド: 基礎代謝 / メンテナンス / ダイエット時 / 増量時 (各 kcal/日)

**チャット #7 で決定した改善 (P3)**:
- 運動量レベルの「× 1.2」等の係数表記 → 数字を非表示にする (受講生に意味不明)
- 「ほとんど運動しない」等の説明だけ表示、係数は内部で持つ

### Step 10d: 減量期間逆算 (`/tools/diet-period`)

**モック**: `/tmp/tools_diet_period.html`

**計算ロジック**:
- 必要な減量 = 現在体重 − 目標体重
- 期間 (週) = 減量 / ペース
- 目標到達日 = 開始日 + 日数

**入力**: 現在体重 + 目標体重 + ペース (kg/週) + 開始日

**出力**: 期間 (週)、到達予定日、進捗バー、週次タイムライン、ペース別アドバイス

**チャット #7 で決定した改善**:
- **P5 タイムライン表示**: 案 α 採用 = 節目週 (1ヶ月/3ヶ月/目標) のみ表示、間は折りたたみ
- **P6 PDF ボタン**: MVP では削除 (Phase 4 で検討)
- **P7 ペース 0.5 デフォルト**: 入れる (推奨値プリセット)

**ペース別アドバイス**:
- ≤ 0.5 kg/週: 「無理の少ない、続けやすい」
- ≤ 1.0 kg/週: 「ややしっかりめ」
- > 1.0 kg/週: 「かなり速い、計画見直しを」

### Step 10e: PFC・カーボサイクル (`/tools/pfc-carb`)

**モック**: `/tmp/tools_pfc_carb.html` (2 段階統合)

**STEP 1: PFC 計算**
```
P (g) = 体重 × 2 / P (kcal) = P × 4
F (kcal) = カロリー × 脂質比率 / 100 / F (g) = F / 9
C (kcal) = カロリー − (P + F) / C (g) = C / 4
```
入力: 体重 + 摂取カロリー + 脂質比率 (20%/25%/30%)

**STEP 2: カーボサイクル配分**
```
係数: HIGH = 1.85倍 / MID = 1.0倍 / LOW = 0.5倍
週合計を 7C (週ベース) に正規化して再配分
```
入力: 月〜日 各曜日のトレ強度 (高/中/低)

**警告**:
- 高 7 日 → 「カロリーオーバー」
- 高 5 日以上 → 「ハイカーボ日が多すぎ、週 2-3 日推奨」
- 低 7 日 → 「筋肉が育たない」

**チャット #7 で決定した改善**:
- **P8 「今日の目安」削除確定** (合意の正典セクション 17、C 案準拠)
- **P9 たんぱく質係数**: 固定 2g (MVP、Phase 4 で 1.6/2.0/2.5 から選択検討)
- **P10 STEP 1 折りたたみ**: 無し (シンプル優先)
