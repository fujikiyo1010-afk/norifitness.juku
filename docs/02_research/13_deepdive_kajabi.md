# 13 深掘り: Kajabi

**URL**: https://kajabi.com/
**料金**: 月額$149〜$399（約23,000〜62,000円）コース運営者向け
**販売形態**: B2B SaaS（コース運営者向け、世界トップシェアの会員制学習プラットフォーム）
**筋肉塾との関係**: **会員制学習プラットフォームの世界標準**、コース構造・ドリップ配信・メンバーポータルの設計参考

---

## 1. サービス概要

| 項目 | 内容 |
|---|---|
| 概要 | 「コース + コミュニティ + マーケ + 決済」をオールインワンで提供する SaaS。世界中の高単価コースクリエイターの定番 |
| 主な対象 | コースクリエイター、コーチ、コンサル、オンラインスクール運営者 |
| メンバーから見ると | 「Kajabi 上で運営されるスクール」に入ってログインし、コース動画・コミュニティ・課題に取り組む |

---

## 2. メンバーサイトの画面構成

### Member Portal の標準機能
- **コース一覧**（受講中・未受講）
- **進捗追跡**（コース毎の完了%・最後に視聴した位置）
- **コミュニティタブ**（運営者がコミュニティを有効化していれば）
- **イベントカレンダー**
- **支払い管理**

### コース内部の構造
| 階層 | 説明 |
|---|---|
| **Product**（製品） | コース全体の容器 |
| **Module**（モジュール） | 章相当 |
| **Lesson**（レッスン） | 個々の動画・PDF・テキスト |
| **Assessment**（評価） | クイズ・アンケート・課題 |

→ 筋肉塾の DB スキーマ `courses → chapters → lessons` と完全一致

---

## 3. 学習継続を促す仕組み ⭐

### ドリップスケジュール
- **公開日指定 or 入会後X日後** にレッスン解放
- 「次が出るのが楽しみ」感を演出
- 筋肉塾の M-3 段階公開と同設計

### 前提クリア（Prerequisite）
- 「前のレッスンを完了しないと次が開かない」設定可
- → 順序通り学ばせる仕組み

### 進捗バー
- コース毎に「あと◯%」表示
- 受講者画面で常時可視化

### 完了マーク
- 各レッスンに「完了」フラグ
- 完了したレッスンはチェック表示

### 通知
- 次のレッスン解放時にメール送信可
- リマインダー機能

---

## 4. 試験・テスト機能

### Assessment 機能
- **クイズ形式**（多肢選択、True/False、記述）
- 合格点設定可
- 何度でも受験可（運営者設定次第）
- 結果は自動採点 + 運営者管理画面で一覧化

→ 筋肉塾の M-8 試験機能と整合（自動採点、何度でも受験可、合否+正答率）

---

## 5. 受講証明書（Certificate）

- コース完了時に **PDF 証明書を自動発行**
- カスタムデザイン可
- 受講者がダウンロード or SNS シェア可

→ 筋肉塾の Should Have「認定書 PDF 発行」と整合

---

## 6. コミュニティ機能

- 独立した「Community」プロダクト（オールインワン内）
- 投稿・コメント・リアクション
- カテゴリ別チャンネル
- ライブイベント連携

→ Skool ほど Reddit 寄りではなく、より「グループ」風

---

## 7. モバイルアプリ

- iOS / Android 専用アプリ
- プッシュ通知
- オフライン視聴（一部）

## 8. UI/UX の特徴

| 要素 | 説明 |
|---|---|
| デザイン | 白基調、運営者ブランドの色を反映可能 |
| メンバーポータル | サイドナビ + メインコンテンツの2カラム |
| カスタマイズ | 各スクールのブランドに合わせてカラー・ロゴ調整可 |
| 学習者画面 | コース → モジュール → レッスンの3階層遷移、パンくず付き |
| 動画プレイヤー | 標準的、再生速度・字幕対応 |

## 9. 筋肉塾への適用ポイント ⭐⭐

### ✅ 採用すべき（高優先）
1. **コース → モジュール → レッスン → 評価の4階層**
   - 既に DB スキーマ一致、UI も追従
2. **ドリップスケジュール = 段階公開**
   - 既に M-3 で予定済、Kajabi の「公開日指定」「入会後X日後」両対応を参考に
3. **前提クリアによる順序強制（オプション）**
   - 「Lesson 1 を完了しないと Lesson 2 が開かない」設定を選択肢に
4. **進捗バー**
   - コース別 / 章別 / 全体 の3階層で
5. **自動 PDF 証明書発行**
   - 卒業時の「修了の演出」機能（Should Have の認定書）
6. **メンバーポータルのサイドナビ + メイン2カラム構造**
   - PC版で採用検討。モバイルは下部タブが優先

### ⚠️ 要調整
- **モバイルアプリ**: 筋肉塾は PWA で先行、ネイティブアプリは将来検討
- **コミュニティ**: 既に M-7 コメント機能で予定済、Kajabi 風の独立コミュニティタブは過剰

### ❌ 真似しないほうがいい
- **オールインワン全部入り**: Kajabi はマーケ・決済も統合だが、筋肉塾は決済・LP 既存活用が決定済
- **複雑な販売ファネル機能**: 筋肉塾の新サイトはあくまで「学習空間」

## 10. アンチパターン

| 由来 | やらないこと |
|---|---|
| Kajabi | 機能を増やしすぎる（全部入り）→ シンプル設計を優先 |
| Kajabi | カスタマイズ自由度高すぎる → 筋肉塾は「のり氏ブランド統一」で一貫性重視 |
| Kajabi | 前提クリアを必須にしすぎる → 復習に戻れない、自由度を一定残す |
