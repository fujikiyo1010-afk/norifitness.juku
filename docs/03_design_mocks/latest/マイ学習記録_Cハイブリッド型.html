C ハイブリッド型の設計意図

背景

  • 2026-05-19 wireframe v2.1 で「ハブ型」(4 カードで他ページへ) として決定
  • 2026-05-20 実装: Phase 2-5b で /my-log としてハブ型 + フラッシュバック + 振り返り一覧サブページ で完成
  • 2026-06-09 モック更新: 「集約型」(統計 + 振り返り 12 件を 1 画面に直接表示) に揺れて、きよむさん的には指示なく構造が変わった状態
  • → きよむさん判断: 「C ハイブリッド型」で行く。両方の良い部分を採用

要素出典状態

破棄した要素

  • XP 表示 (リサーチで議論、ゲーム感を避けるため不採用)
  • ランキング (比較ストレス回避)
  • カレンダーヒートマップ (プレッシャー源)

未確定 / Phase 4 候補

  • マイノート (検討余地、最初は振り返りに含める)
  • バッジ表示 (12 個程度に絞った後、別画面 or プロフィール画面?)
  • 下部タブ「記録」と本画面 (ホーム 4 機能ブロックの「マイ学習記録」) の 動線重複問題 — 本画面に両方からアクセスする設計 OK? それとも下部タブは別画面 (体組成 + 月次添削 のハブ) にする?
