🗺 画面の役割

  • 受信箱 (3a) で 1 件選択した後の 個別作業画面
  • 17 項目のテキスト回答を読みながら、1 本の動画で返信を作る

📐 構成 (上から)

  • 受講生プロフィール: 名前 + 入会日 + 過去回数 + 経過日数
  • 17 項目回答 (フラット): Q1 → Q17 を縦 1 列、質問 + 点数 + テキスト回答
  • 過去返信履歴: この受講生に過去何を言ったか (繰り返し回避)
  • 動画返信エリア: 2 ボタン (ブラウザ録画推奨 / ファイルアップロード)
  • 操作バー: sticky bottom、下書き保存 / 送信終了 / 送信して次へ

🎥 動画返信の 2 方式

方式 1: ブラウザ録画 (推奨、案 V)

  • ボタン押下 → 録画モード画面 (admin_monthly_record.html) に遷移
  • 左に 17 項目テキスト / 右にカメラプレビュー の分割レイアウト
  • MediaRecorder API でブラウザネイティブ録画
  • 録画停止 → プレビュー → 送信 → Supabase Storage 自動アップロード

方式 2: ファイルアップロード

  • tldv ダウンロード / スタジオ撮影 / Zoom 等で録った動画を後からアップロード
  • 高画質にこだわりたい時、屋外で撮りたい時用

💾 動画ファイル仕様

  • 形式: webm (ブラウザ録画) / mp4 / mov (アップロード)
  • 推奨: 5 分以内、80MB 以下 (超過時は警告のみ)
  • 保存先: Supabase Storage、永久保存
  • 受講生退会時はアクセス権限のみ切る (データは残す)

📝 17 項目回答エリアの設計

  • 1 列フラット: カテゴリ分類なし、Q1 → Q17 を縦に並べる
  • Q + 点数 + テキスト回答 が 1 ブロック
  • テキスト回答は 左ボーダー (緑) で視覚的に区切り、読みやすく
  • Q12 など低スコアは控えめにグレー表示 (赤強調しない、責めない)

🔘 操作ボタン

  • [下書き保存]: 録画済みだけど送信していない状態を保持
  • [送信して終了]: 送信 → 受信箱 (3a) に戻る
  • [送信して次へ →]: 送信 → 受信箱の次の未返答に自動遷移 (集中作業リズム)
