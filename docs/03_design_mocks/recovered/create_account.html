画面の役割

  • 申請承認後、メールで受け取った「アカウント有効化リンク」から遷移
  • 受講生がパスワードを設定して、アカウント作成を完了する画面

構成 (上から)

  • ヘッダー: 「アカウント作成」(戻るなし、メール経由の直接遷移)
  • アイコン (ユーザー) + 「パスワードを設定」+ 説明
  • フォーム 4 項目:
    • 氏名 (固定表示): 管理画面登録名、編集不可、鍵アイコン + 注釈
    • メアド (固定表示): 申請時メアド、編集不可、鍵アイコン + 注釈
    • パスワード (新規入力)
    • パスワード確認
  • パスワード条件チェックリスト (リアルタイム反映)
  • 「アカウントを作成する」ボタン

確定方針 (2026-06-10)

氏名 = 固定 (管理画面で発行時に入力した名前で表示、変更不可)
理由:
・LINE と同じ名前で統一 → 管理画面/受講生 UI で識別ブレなし
・月次返信動画で のり氏が氏名で呼ぶ → アプリも同じ名前で一貫性
・「ニックネーム」フィールドは廃止 (コミュニティ機能なし、シンプル化優先)

メアド = 固定 (申請時メアドで自動表示、編集不可)
・きよむさんが承認したメアドと一致前提
・変更したい場合は LINE で連絡 → きよむさんが管理画面で更新

動線

  • 申請フォーム (/request) → きよむさん承認 → メール受信 → リンククリック → この画面
  • パスワード設定 + 「アカウントを作成する」 → 自動ログイン
  • 初回起動 → オンボーディング 8 ステップ (カルテ / 目標 / 体組成 / プロテイン発送先 必須)
