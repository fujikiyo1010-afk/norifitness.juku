# 15 深掘り: Salesforce Trailhead

**URL**: https://trailhead.salesforce.com/
**料金**: 無料（Salesforce 認定試験は別途有料）
**販売形態**: Salesforce のオウンドメディア・教育プラットフォーム
**筋肉塾との関係**: **バッジ獲得型ゲーミフィケーション学習の世界標準**、企業向け学習プラットフォームの教科書 ⭐⭐⭐

---

## 1. サービス概要

| 項目 | 内容 |
|---|---|
| 概要 | Salesforce が運営する無料の学習プラットフォーム。CRM スキルだけでなくビジネス基礎・データリテラシー・対人スキルまで網羅 |
| 主な対象 | Salesforce エンジニア・管理者・ユーザー（だが誰でも参加可） |
| 強み | **バッジ×ランク×Trailmix のゲーミフィケーション**、「学びが楽しい」を体現 |

---

## 2. 学習コンテンツの階層構造

| 階層 | 説明 |
|---|---|
| **Module（モジュール）** | 個別トピックの学習単位、複数のレッスン+クイズで構成 |
| **Project（プロジェクト）** | 実際に手を動かす実習タイプ、より長め |
| **Trail（トレイル）** | 関連 Module/Project を集めた学習パス「キャリア◯◯になる道」 |
| **Trailmix（トレイルミックス）** | カスタム学習プレイリスト、自分で組める or 他者作成を利用可 |

→ 「点（Module）→ 線（Trail）→ プレイリスト（Trailmix）」の階層化

---

## 3. ゲーミフィケーション設計

### バッジ（Badge）
- Module・Project 完了で **1バッジ取得**
- 個別のグラフィックデザインがあり、視覚的に蓄積されていく
- 「コレクション欲」を刺激

### ポイント（Points）
- Module/Project 完了で +ポイント
- バッジ数×平均ポイント = 累計ポイント

### Trailblazer Rank（ランク）

| Rank | 必要バッジ | 必要ポイント |
|---|---|---|
| Scout（スカウト） | 0 | 0 |
| Hiker（ハイカー） | 1 | - |
| Explorer（エクスプローラー） | - | - |
| Adventurer（アドベンチャラー） | - | - |
| Mountaineer（マウンテニア） | - | - |
| Expeditioner（エクスペディショナー） | - | - |
| **Ranger（レンジャー）** | 100 | 50,000 |
| Double Star Ranger | 200 | 100,000 |
| Triple Star Ranger | 300 | 150,000 |
| ... All Star Ranger まで | - | - |

→ **明確なゴール（Ranger）+ その先の継続目標（Star 系）** の二段構え

### Confetti & Celebration
- バッジ取得時に画面に **紙吹雪エフェクト**
- 「小さな成功」を派手に祝う

### Trailblazer Community
- ランク・バッジが Salesforce イベントで認知される（リアル世界の称号）
- 公式コミュニティでプロフィール表示
- 採用市場でも「Ranger 取得者」が評価対象になる

---

## 4. 学習体験のコア

### Module の構成
1. **イントロ動画/テキスト**
2. **複数の Unit（小ユニット）**
3. **各 Unit に学習コンテンツ + クイズ**
4. **最後の Hands-on Challenge（実技）**
5. **合格でバッジ獲得**

### Quiz（クイズ）の特徴
- 多肢選択
- **不合格でも何度でも挑戦可**
- ヒント表示あり
- 「ここを読み返してみよう」誘導

### Hands-on Challenge
- Salesforce 環境を実際に触る課題
- 自動採点（環境チェック）
- 失敗時は具体的なフィードバック

## 6. 筋肉塾への適用ポイント ⭐⭐⭐

サクセスラーニング型の世界標準であり、筋肉塾の参考価値は極めて高い。

### ✅ 採用すべき（強推奨）
1. **Module = Chapter、Project = 実技課題、Trail = コース全体** のマッピング
   - 既に DB スキーマと整合
2. **バッジ獲得 = 章完了 or 試験合格時**
   - 視覚的なグラフィックバッジを章ごとに用意
   - 「コレクション」を促す
3. **ランク制度（Skool のレベルと統合可能）**
   - 「初心者 → 中級者 → 上級者 → マスター → グランドマスター」等
   - 必要なポイント/バッジ数でランクアップ
   - 明確なゴール（マスター）+ その先（グランドマスター）の二段構え
4. **Confetti 等の「小さな成功」を派手に祝う**
   - 「できた！」ボタン押下時に紙吹雪エフェクト
   - 試験合格時に祝福アニメ
   - **「実施完工率向上」に直結する小さなご褒美**
5. **クイズの「何度でも挑戦可 + ヒント表示」**
   - 既に M-8 試験機能で「何度でも受験可」、合格点(80%) 採用予定
   - 不合格時の「ここを見直そう」誘導を追加検討
6. **公開プロフィール（ランク・バッジ表示）**
   - 卒業後の SNS シェア（Should Have）と整合
   - 「Ranger 取得者」相当の認証システム

### ⚠️ 要調整
- **バッジの量**: Trailhead は数百個、筋肉塾は20〜50個程度に抑える（数が多すぎると価値希薄化）
- **Hands-on Challenge**: 筋肉塾は実際の体の変化が「実技」相当、自動採点は難しい → 自己申告 + 写真投稿で代替
- **公開プロフィール**: 受講生の個人情報リスク、設定で ON/OFF 切替

### ❌ 真似しないほうがいい
- **学習パスの数を膨らます（Trail × Trailmix）**: 筋肉塾は5大カテゴリ固定で十分
- **採用市場連動**: 筋肉塾の卒業生証は SNS シェア程度に留める

---

## 7. アンチパターン

| 由来 | やらないこと |
|---|---|
| Trailhead | バッジを増やしすぎて価値希薄化 |
| Trailhead | ランクで「初心者」を晒す → 筋肉塾はランクを誇示型でなくモチベ型に |
| Trailhead | Hands-on の難易度上げすぎて挫折 → 筋肉塾の試験は **基礎理解を確認する** 程度に |
