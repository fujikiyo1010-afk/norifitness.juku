3 案 比較

項目 A ハブ型 B 集約型 C ハイブリッド型
フラッシュバック ✅ あり ❌ なし ✅ あり
今月の学習統計 ❌ なし ✅ あり ✅ あり
累計進捗バー ❌ なし ✅ あり ✅ あり
振り返り表示 ハブカード経由で別画面 この画面に直接 3 件 + さらに見る この画面に直接 3 件 + すべて見る
将来機能 (ブクマ/実践/完了履歴) 2x2 で 4 ハブカード ❌ なし 3 列コンパクトで 3 個
画面の縦長さ 短い (フラッシュバック + 2x2) 中 長い (全部入り)
既存実装との整合 ✅ 完全一致 (改修不要) ❌ 完全改修必要 部分改修必要
受講生が一目で得る情報量 少 中 (振り返り中心) 多 (統計 + 振り返り + 将来機能)
「学習感」の演出 普通 強い (統計で蓄積感) 強い + 暖かい (フラッシュバック)

各案のメリット・デメリット

案メリットデメリット
A ハブ型 シンプル / 実装と一致 (改修不要) / フラッシュバックで暖かみ 統計なし → 学習の蓄積感が弱い / ハブカードを押さないと振り返り見えない
B 集約型 統計で蓄積感 / 振り返りが直接見える フラッシュバックなし (暖かみ弱) / 将来機能 (ブクマ・実践等) の入口なし / 実装改修大
C ハイブリッド型 全部入り (フラッシュバック + 統計 + 振り返り直接 + 将来機能ハブ) 画面が縦に長い / 情報多すぎ?

確認ポイント

  • 「学習感の演出」を優先するなら B か C (統計が魅力)
  • 「実装改修を最小化」したいなら A
  • 「全部入り」が筋肉塾の世界観に合うなら C
  • 「シンプル過ぎず、複雑過ぎず」のバランスを取るなら C (ただし縦長注意)
