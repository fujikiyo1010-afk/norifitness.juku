🗺 録画モードの 3 状態

  • 状態 1: 録画準備中 — カメラ許可後、録画開始待ち。左に 17 項目、右に自分のプレビュー
  • 状態 2: 録画中 — 録画中インジケータ + タイマー、左をスクロールしながら話せる
  • 状態 3: プレビュー — 録画完了後、再生確認 → 採用 or 録り直し

🎬 状態遷移フロー

個別画面 (3b) 「🎥 ブラウザで録画する」
→ 状態 1: 録画準備中 (このページ、カメラ許可ダイアログ → 待機)
→ 録画ボタン押下 + 3 秒カウントダウン
→ 状態 2: 録画中 (REC マーク + タイマー)
→ 停止ボタン押下
→ 状態 3: プレビュー (確認 → 採用)
→ 個別画面 (3b) に戻る + Supabase Storage に自動アップロード開始
→ 「送信して終了」or「送信して次へ」を押下

📐 レイアウト設計

  • 左右 50:50 分割
  • 左側 (テキスト): 受講生プロフィール + 17 項目、スクロール可能
  • 右側 (カメラ): ダーク背景でカメラに集中、録画コントロール
  • ヘッダー: 状態バッジ (待機 / REC / 完了) + 終了リンク

🎥 録画中の挙動 (実装仕様)

  • カメラ枠: 録画中は赤縁取り + 影、視覚的に「録画中」と分かる
  • タイマー: カメラ枠左上に赤ドット点滅 + 経過時間 (mm:ss)
  • 左側ハイライト: 現在話している項目を黄色背景で示す? (オプション、未確定)
  • 一時停止: 録画を一時停止できる (誤動作対策)
  • 5 分超過警告: タイマーが赤くなる、強制停止はしない

💾 録画完了後の保存フロー

  • 状態 3 で「採用」押下 → 個別画面 (3b) に戻る
  • 戻った後、3b の動画返信エリアに「録画済み」プレビューが表示される
  • Supabase Storage へのアップロードは バックグラウンドで非同期実行
  • 「送信して終了 / 次へ」押下時にアップロード完了を待つ

⚠️ 未確定 / 議論ポイント

  • カメラ位置: PC 内蔵カメラ / 外部 Web カメラ どちらも対応
  • マイク選択: 複数マイク接続時に選べる UI
  • 画質設定: 720p 固定 / 1080p 選択可 (Supabase ストレージとのバランス)
  • 録画中にページ離れた時の警告 (beforeunload)
  • 左側ハイライト (話してる項目を強調) はオプション機能か必須か
  • 「録り直す」時、既存動画は破棄? 確認ダイアログ要?

🛠 技術仕様 (実装メモ)

  • 録画 API: MediaRecorder API (ブラウザネイティブ)
  • 対応ブラウザ: Chrome 49+ / Safari 14.1+ / Edge / Firefox
  • 録画形式: webm (Chrome/Edge/Firefox) / mp4 (Safari)
  • 解像度: 720p デフォルト (constraints で指定)
  • アップロード: Supabase Storage upload() または chunked upload
  • 非同期処理: useEffect でカメラ起動、useRef で MediaRecorder インスタンス保持
