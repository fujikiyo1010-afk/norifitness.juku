# 17 デザイン素材まとめ v2（サクセスラーニング型・最終版）

**作成日**: 2026-05-19
**目的**: 6社の深掘り + UIパターンを **筋肉塾の8画面構成** にどう落とし込むかを最終確定
**前提**: [handoff_2026-05-19.md](../00_premises/handoff_2026-05-19.md) で確定した MVP 12機能 / 16テーブル / サイトマップ

**対象6社**: Schoo / SHElikes / Skool / Kajabi / Duolingo / Trailhead

---

## 🎯 統合コンセプト

> **「学ぶこと自体が楽しい・続けたくなる」サクセスラーニング型プラットフォーム**

リサーチから抽出した4本柱:

1. **小さな成功を派手に祝う**（Duolingo / Trailhead 紙吹雪文化）
2. **連続記録で離脱を阻止**（Duolingo Streak 心理設計）
3. **バッジ・レベルで蓄積感**（Skool / Trailhead ゲーミフィケーション）
4. **目標を起点に学びを構造化**（SHElikes コーチング × トレイル思想）

## 2. ゲーミフィケーション機能の設計（横串）

### 統合ポイント・レベル設計

| アクション | XP | 採用元 |
|---|---|---|
| レッスンに「できた！」 | +10 XP | Skool |
| コメント投稿 | +5 XP | Skool |
| コメントに👍をもらう | +1 XP | Skool |
| 試験合格（初回） | +50 XP | Trailhead |
| 試験合格（再挑戦） | +20 XP | - |
| 章完了 | +100 XP（バッジ獲得） | Trailhead |
| 7日連続記録達成 | +30 XP | Duolingo |

### レベル設計（筋肉塾オリジナル命名）

| Lv | 名称 | 必要XP | 解放されるもの |
|---|---|---|---|
| 1 | 入塾生 | 0 | スタート |
| 2 | 探求者 | 200 | 同期内ランキング閲覧 |
| 3 | 修行者 | 800 | プロフィールのカスタムアバター |
| 4 | 達人 | 2,000 | 卒業バッジ獲得資格 |
| 5 | 師範 | 5,000 | グランドマスター挑戦権 |
| ∞ | グランドマスター | 10,000+ | 永久会員レア称号 |

### ストリーク設計

```
[🔥 連続記録]
- 1日1アクション（できた！ / コメント / 視聴1分）で +1日
- Streak Freeze: 月2回まで「忘れ」を自動救済
- 7日 → 黄色炎、30日 → オレンジ、100日 → 青炎（レア）
- マイページ・上部ヘッダーに常時表示
```

## 3. アンチパターン統合（やらないこと）

リサーチ全社から横断的に「避けるべき」を抽出:

| 由来 | やらないこと |
|---|---|
| Duolingo | Streak を絶対視 → 失った時の自責で離脱増 |
| Duolingo | 体力制（Hearts）→ 罰則的すぎ、筋肉塾には合わない |
| Skool | 全期間ランキングで下位者を可視化 → 上位のみ表示 |
| Skool | ポイントが「いいね」依存 → 投稿しない人が不利、行動も加点 |
| Trailhead | バッジを増やしすぎ → 価値希薄化、20〜50個に絞る |
| Kajabi | 全部入り機能 → シンプル設計優先 |
| Kajabi | 前提クリア必須 → 復習に戻れない、自由度残す |
| SHElikes | 強制コーチング → 筋肉塾は LINE 飛ばしで柔軟 |
| SHElikes | コース選択肢多すぎ → 完工率低下 |
| Schoo | コメント中心 → 学習集中度低下、コメントは副次 |

## 4. 採用しないが「面白かったので頭の隅に」候補

### Duolingo Hearts → ❌不採用
- 体力制限・課金で解消は筋肉塾と合わない

### Trailhead 採用市場連動 → ❌不採用
- 筋肉塾卒業証は SNS シェア程度

### Skool Gems → ❌不採用
- 仮想通貨的で複雑すぎ

### Skool Map（メンバー地図）→ 🤔保留
- プライバシーリスク、希望者のみON

### Trailhead Trailmix（カスタム学習プレイリスト）→ 🤔保留
- Should Have で「マイ学習プラン」として将来検討

## 5. 既に実装済み機能との整合性確認

別チャットで既に実装された機能 vs このリサーチの提案:

| 実装済み | このリサーチでの提案 | 整合性 |
|---|---|---|
| 19テーブル + RLS + Storage 3バケット | DB 構造は変更不要、本リサーチは「UI設計の素材」 | ✅ |
| 認証フロー（招待→パス設定→ログイン） | sitemap_draft.md と整合 | ✅ |
| /admin/invitations | sitemap_draft.md と整合 | ✅ |
| Resend メール送信 | 招待メールに使える | ✅ |

→ 本リサーチは **既存実装を壊さず、UI/UXの拡張案として位置付け** できる
