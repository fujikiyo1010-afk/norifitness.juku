# 社長への確認事項リスト

**作成日**: 2026-05-18
**用途**: フェーズ0 抽出 + エキスパ調査で**取れなかった**・**確認が必要**な項目を集約。きよむさんが社長に直接ヒアリングする際の質問集。
**優先度の凡例**: 🔴 ブロッカー（フェーズ1で技術選定や Supabase 設計に直結） / 🟡 重要（MVP 範囲に影響） / 🟢 後でも可（MVP 後の運用設計）

## 🔴 Tier 1: ブロッカー（フェーズ1着手前に必要）

### A. 現状の決済・会員管理の実態

| # | 質問 | 背景 |
|---|---|---|
| A-1 | 現在、受講生は**どこで決済**している？（Stripe? UnivaPay? エキスパ決済? PayPay? 銀行振込? その他?） | 新サイトの決済選定に直結 |
| A-2 | 決済プロバイダの管理画面にきよむさん/AI はアクセスできる？ | データ移行・連携可否の判断 |
| A-3 | 受講生のマスタデータ（氏名・連絡先・入会日・プラン）は**どこに保管**されている？ | エキスパの会員一覧は「招待会員・期限切れ」で実質未使用と判明。実体の所在を確認 |
| A-4 | 月額課金 or 買い切り or 両方ある？プランは何種類？それぞれの価格は？ | Supabase テーブル設計の最重要項目 |
| A-5 | 受講生が**退会**するとき、どういうフローを通る？（自動 or 手動 / メール or LINE） | 解約 UX 設計 |
| A-6 | **銀行振込割引（¥30,800 OFF）は新サイトでも継続予定でいい？** | きよむさん「継続」方針、社長確認 |
| A-7 | **「2日間限定」等の時限価格表示は新サイトでも継続予定でいい？** | きよむさん「継続」方針、社長確認。MVP に含めるか判断材料 |
| A-8 | **リバウンド保証の規定（何日以内に何の条件で返金）の正確な仕様** | 新サイトの DB 設計に必要（返金条件追跡機能） |

## 📋 ヒアリング進行の提案

きよむさんが社長と話す時の**進め方提案**:

### 1回目（30〜45分目安）：Tier 1 ブロッカー A〜D
新サイトの**根幹設計**に必要な情報。これが分からないと技術スタックも Supabase 設計もできない。  
→ A-1〜A-5、B-1〜B-3、C-1〜C-3、D-1〜D-3 を中心に。

### 2回目（30〜45分目安）：Tier 2 重要 E〜H
ビジネス数字とスコープ判断。MVP の範囲を確定する材料。
→ E-1〜E-5、F-1〜F-3、G-1〜G-5、H-1〜H-3。

### 3回目（フェーズ2以降）：Tier 3 + メタ質問
デザイン方針と思想の言語化。
→ I〜L 全般。

### 📝 補助ツール
- 質問は**事前に文書化して社長に送る** → 社長が考える時間を持てる
- 1on1 で口頭ヒアリング → きよむさんが文字起こし → 私が解析
- 録音/Notion 共有も検討（社長確認用サマリーと相性が良い）
