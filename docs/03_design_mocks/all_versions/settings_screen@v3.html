🗺 画面の役割

  • 下部タブ「⚙ 設定」から遷移
  • アカウント情報・連携・規約・ログアウト等の集約
  • シンプル 3 セクション (追加候補のプラン情報/学習設定/表示設定 + 通知設定 は不採用)

📐 構成 (2026-05-24 更新)

  • 【アカウント】: プロフィール / メールアドレス / パスワード変更
  • 【連携】: LINE 連携 (アカウント結合)
  • 【サービス】: 利用規約 / プライバシー / ヘルプ / お問い合わせ / アプリ情報
  • [ログアウト]: 白背景・グレー文字 (赤は使わない)
  • 退会はお問い合わせ経由のみ (画面にボタンなし、フッターに案内)

🔕 通知設定セクションを削除した理由 (重要)

  • プッシュ通知 + LINE 配信 = 両方とも強制送信 (きよむさん判断 2026-05-24)
  • 受講生がオフにする手段はアプリ内に存在しない
  • 運営側として「絶対送らせたい」方針を貫徹
  • ※ OS 側の通知許可 / LINE ブロックは受講生が能動的にできる範囲 (技術的限界)
  • これに伴い「通知」セクション自体を削除 → 設定画面が 3 セクションに簡素化

🔄 動的挙動

  • 各項目タップで詳細画面 or アクション
  • 「プロフィール」タップ → プロフィール画面へ遷移
  • 「ログアウト」タップ → 確認モーダル → ログイン画面へ

🎨 デザイン方針

  • シンプル list 形式、カード角丸 12px
  • セクション間に余白
  • スイッチは ティール緑 (On 時) / グレー (Off 時)
  • 赤色は使わない (ログアウトもグレー、責めない・不安にさせない)

📝 退会フロー (お問い合わせ経由のみ)

  • 受講生が「お問い合わせ」をタップ
  • お問い合わせフォーム (メール or LINE)
  • のりfitness 側で対応 (理由ヒアリング + データ削除手続き)
  • 退会ボタンを置かない = 衝動的な退会を防ぐ + 引き止めの機会
