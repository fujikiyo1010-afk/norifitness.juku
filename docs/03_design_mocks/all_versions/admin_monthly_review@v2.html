🗺 画面の役割

  • 受信箱 (3a) で 1 件選択した後の 個別作業画面
  • 受講生 1 人の月次報告を確認 → 動画録画 → コメント → 送信、を完結させる
  • シーン 1 (月初の集中作業) で 10 件まとめ返信する時の主役画面

📐 構成 (上から)

  • 受講生プロフィール: 名前 + 入会日 + 過去回数 + 危険信号フラグ
  • 17 項目回答: カテゴリ別 2 列レイアウト (PC 横幅活用)
  • 過去返信履歴: この受講生に過去何を言ったか (繰り返し回避)
  • 動画アップロード: ドラッグ&ドロップ + ファイル選択
  • コメント入力: テンプレ選択 + 自由入力 + 文字数カウント
  • 操作バー: 下書き保存 / 送信終了 / 送信して次へ (sticky bottom)

🎬 動画アップロード仕様

  • 形式: mp4 / mov / webm
  • 5 分以内、80MB 以下 推奨 (超過時は警告のみ、ブロックはしない)
  • 解像度の自動圧縮なし (案 γ: 運用ルールで縛る)
  • 保存先: Supabase Storage (Pro プラン契約後)
  • Phase 2 で MediaRecorder API による ブラウザ録画 も追加 (両対応)

🔘 「送信して次へ」併用ボタン (確定)

論点 1 対応: 2 ボタン併置で、リズム良く回したい時は「次へ」、1 件で休憩なら「終了」を選べる。
  • 「送信して次へ」: 送信 → 受信箱 (3a) の次の未返答に自動遷移
  • 「送信して終了」: 送信 → 受信箱 (3a) に戻る
  • 「下書き保存」: 送信せず保存 (途中保存)

💬 コメントテンプレート (MVP 含む)

  • 過去のコメントから自動分類 (食事/運動/休息など)
  • テンプレ選択 → テキスト挿入 → 編集
  • 「新しいテンプレを保存」で運用しながら蓄積

🎨 受講生 UI との関係

  • 色: ティール緑 #00897b を共通
  • 添削色: 黄 (受講生 UI と同じ)、危険信号は赤を追加 (運営側のみ)
  • 角丸: 8-12px (受講生 14px より少しシャープ、情報密度高い PC 想定)
  • レイアウト: 17 項目を 2 列にして 1 画面に収める (PC 横幅活用)

  • 下書き保存後、いつ自動破棄するか (7 日? 永久?)
  • 送信後の受講生通知方法 (LINE 配信? アプリ内のみ? 両方?)
