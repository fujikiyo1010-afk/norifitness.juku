✅ 反映した仕様

  • 入力 UI = 案 2-D ハイブリッド: 大表示 (中央 32px Inter) + ステッパー 4 ボタン + 大表示タップで直接入力モード
  • 増減ステッパー: 体重 −0.5/−0.1/+0.1/+0.5、ウエスト/首回り −1.0/−0.5/+0.5/+1.0
  • 前回比: 「前回から −0.2 kg」グレー #888 (変動方向問わず同色)
  • 体脂肪率: 自動算出 (入力不可)、ウエスト/首/身長から
  • ⓘ ツールチップ: 体脂肪率の補足「アメリカ海軍式で算出」(ホバー表示)
  • 日付選択: ヘッダー帯の日付がタップ可 (カレンダーUIで過去日遡及)
  • 「今日」の文字色: 緑 → 黒 #1a1a1a (静かなトーン統一)
  • delta 色: 全部グレー (責めない、催促しない)

📊 ミニ折れ線グラフ (新規)

  • SVG で軽量に描画 (外部ライブラリ不要)
  • 直近 7 日の体重のみ
  • 未記録日 (5/18) は色違い (グレー)
  • 今日のポイントは少し大きめ + 白枠

📅 履歴リスト

  • 直近 7 日 (実装時は 30 日まで)
  • 体重と体脂肪率を縦揃え (130px 固定幅)
  • 未記録日もグレーで表示 (飛ばさない、責めない)
  • delta は前回比、サイン (+/−) で増減を示すが色は統一グレー

🔗 連動の流れ

  • 記録するボタン → body_metrics テーブルに保存
  • 保存後、トースト「✓ 保存しました」のみ表示 (励まし文言なし)
  • 目標シート ① は最新日付の recorded_at 行を参照、リアルタイム反映
  • 変化を見る画面でグラフのデータ源として使用

🗄 DB スキーマ (新規)

  • body_metrics: id / user_id / recorded_at / submitted_at / weight / waist / neck / body_fat_pct / created_at / updated_at
  • UNIQUE 制約 (user_id, recorded_at) で同日重複防止 (上書き更新)
  • 身長は users / profiles の既存値を参照

🚫 実装しないもの (責めない方針)

  • ホーム CTA「今日の体組成を記録する」リマインダー
  • プッシュ通知
  • 朝の挨拶への組み込み
  • 連続記録 (Streak) 演出
  • 節目演出 (10 日連続! 等)
  • 励まし文言 (「ナイス減量!」等)
  • 遡及入力ラベル (「事後入力」マーク等) ※ submitted_at は内部メタデータのみ

📋 配置

  • 下部タブ「記録」内に「📊 体組成記録」を新設
  • 記録タブのトップ or サブメニュー 1 項目として配置
  • 受講生は自発的にここを開く動線のみ
