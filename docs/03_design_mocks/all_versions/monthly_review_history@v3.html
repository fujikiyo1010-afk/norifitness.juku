🗺 全体構成

  • ヘッダー: ← 戻る + 「月次添削 履歴」 (シンプル)
  • ブロック A: 最新月の 4 状態カード (A 未記入 / B 記入中 / C 提出済み・返信待ち / D 返信届いた) ← 別チャット指示反映
  • ブロック B: 推移ビジュアライズ
    • B-1a: レーダーチャート (案 1)
    • B-1b: 横棒バー (案 2)
    • B-2: 月次推移グラフ (折れ線、共通)
  • ブロック C: 月別ログ一覧 (タップで詳細遷移)

※ ブロック A は 4 状態を比較できるよう縦並びで全部見せています。実装時は受講生の現在状態に応じて 1 つだけ表示。
※ B-1a と B-1b も比較表示、実装時はどちらか 1 つ採用。

📋 ブロック A: 4 状態カードの判定

状態条件
A 未記入monthly_audits レコードなし、または items_filled_count = 0
B 記入中submitted_at IS NULL かつ items_filled_count > 0
C 提出済みsubmitted_at IS NOT NULL かつ nori_video_published_at IS NULL
D 返信届nori_video_published_at IS NOT NULL

重要原則: 「提出ボタンを押した瞬間が、初めて のりfitness に通知される境界線」。記入中 (状態 B) は受講生個人の作業領域、のりfitness 側からは一切見えない。

⚖️ レーダー vs バー の比較ポイント

案 1: レーダーチャート

○ 強み: 6 軸が一目で見渡せる ・ 「バランス」の概念を視覚化 ・ 派手で印象的
× 弱み: 凹んだ軸が "弱点" として強く出る ・ 数値の正確な比較がしづらい ・ ダイエット文脈に若干合わない (元々ステータス系ゲーム由来)

案 2: 横棒バー

○ 強み: 数値が正確に読める ・ 縦線 | で前回値も併記でき比較しやすい ・ 上から順に読める (情報の取りやすさ ◎)
× 弱み: 派手さなし ・ 「バランス」概念は伝わりにくい ・ 6 行縦に並ぶので縦幅は食う

→ 「責めない」方針 + 数値の正確さ を重視するなら案 2 (バー) 寄り、
→ 視覚インパクト + 振り返りの楽しさ を重視するなら案 1 (レーダー) 寄り。
