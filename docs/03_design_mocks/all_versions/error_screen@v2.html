🗺 画面の役割

  • 通信失敗・サーバーダウン・404 等の異常時に表示
  • 焦らせず、寄り添う口調で「もう一回試してね」
  • 1 つの画面構造で 3 状態を吸収 (実装シンプル)

📐 共通構造

  • キャラ画像: 96px、状況に応じてポーズ違い
  • タイトル: 1-2 行、状況の説明 (フラットに)
  • メッセージ: 1-2 行、次の行動の案内
  • CTA: 1 ボタン (再読み込み or ホームに戻る)
  • ヘッダー・下部タブはなし (集中させる)

📝 メッセージテンプレート集

キャラ: 困り顔 😅
タイトル: 「うまく繋がらないみたいです」
メッセージ: 「電波の届く場所でもう一度試してみてください」
CTA: 🔄 再読み込みする

キャラ: 申し訳なさそう 🙇

CTA: 🔄 再読み込みする

メッセージ: 「リンクが古い可能性があります」
CTA: 🏠 ホームに戻る

🎨 デザイン方針

  • 背景: 白 (装飾なし、シンプル)
  • 赤色は使わない (責めない、不安にさせない)
  • キャラ 96px (空状態より少し小さめ、控えめ)
  • CTA = ティール緑 + 角丸 12px
  • 下部タブ非表示 (このエラーから抜ける動線は CTA 1 つだけ)

📝 メッセージのトーン (共通)

  • 「エラー」「失敗」のような硬い言葉を避ける
  • 「みたい」「ようです」で柔らかく
  • 「申し訳ございません」より「うまくいかないみたいです」
  • 受講生のせいにしない (「電波が悪い」とは言わず「繋がらない」と中立に)

🤖 実装ルール

  • 1 つの React コンポーネント <ErrorScreen type="connection|server|notfound" />
  • type に応じてキャラポーズ + メッセージ + CTA が切り替わる
  • キャラポーズ画像のパスは type ベースで自動選択
  • MVP では絵文字でプレースホルダー、Phase 2 で実画像差し替え
