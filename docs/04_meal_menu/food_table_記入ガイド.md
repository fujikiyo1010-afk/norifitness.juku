# food_table 記入ガイド（のり監修用）

P4-b（食材の自動計算）で使う「ミニ成分表」を、のり氏監修で仕上げるためのシートです。
`food_table_記入シート_雛形.csv` を Excel / Google スプレッドシートで開いて編集してください。

## 使い方（3ステップ）
1. `food_table_記入シート_雛形.csv` を開く（34品を八訂の値で仮入力済み）。
2. 各列を のり監修で確認・修正（下の表を参照）。品を足す・消すのも自由。
3. 仕上がったら私に渡してください。DB（food_table）へ取り込みます（dev→prod）。

## 列の意味

| 列 | 意味 | 記入のヒント |
|---|---|---|
| `name` | 食品名（受講生の検索・チップに出る名前） | 例「もち麦飯」「全卵」 |
| `aliases(かな・別名/区切り)` | 検索でヒットさせたい別名。**半角スラッシュ `/` 区切り** | 例「なっとう/ナットウ」。空でも可 |
| `unit_type(weight/count)` | **weight=グラムで数える** / **count=個数で数える** | ごはん・肉・野菜=weight／卵・納豆・バナナ=count |
| `base_qty` | 成分値の基準量。weightは`100`(=100gあたり)／countは`1`(=1個あたり) | 触らなくてOK（型に応じて自動） |
| `default_qty` | 投稿画面を開いた時の初期表示量 | weight=よく食べるグラム(白米180 等)／count=`1` |
| `step_qty` | ＋−ボタン1回の刻み | weight=`10`(10g刻み)／count=`1`(±1) |
| `unit_label` | 画面に出す単位 | 'g' '個' 'パック' '本' '杯' '缶' |
| `kcal / protein_g / fat_g / carb_g` | **base_qtyあたり**のPFC | weight=100gあたり／count=1個あたり |
| `fiber_g` | 食物繊維(任意・当面は表示しないが保持) | 空でも可 |
| `sort_order` | チップ/一覧の並び順（小さいほど上） | よく使う品を小さい番号に |
| `is_active` | 有効なら `true`、隠すなら `false` | 基本 `true` |

## 大事なポイント
- **count型は「1個あたりの値」を入れます**。雛形では卵・納豆・バナナ・アボカド・サバ缶・プロテインを count に仮変換済み（1個/1パック/1杯あたりに換算）。数値が実感と違えば のり監修で直してください。
- weight型の `default_qty` は「その食材をだいたい何グラム食べるか」。受講生はそこから10g刻みで微調整します（固定ではない）。
- 迷ったら **weight型のまま**でも動きます（受講生がグラムで入れる）。count型は「個数で数えるほうが自然」な品だけで十分です。
- 計算式（アプリが自動でやること）: weight = 値 × 量 ÷ 100 ／ count = 値 × 個数。のりが入れるのは**基準量あたりの値だけ**でOKです。

## 種データの出所
`food_table.py`（日本食品標準成分表 八訂 準拠・可食部100gあたり）の34品を雛形に反映しています。
