# 12 深掘り: Skool

**URL**: https://www.skool.com/
**料金**: 月額$99（約15,000円）でコミュニティ運営者向け、各コミュニティの会員は無料 or 有料（運営者設定）
**販売形態**: B2B SaaS（コミュニティ運営者へ）+ B2C（メンバー視点）
**筋肉塾との関係**: **コミュニティ+学習一体型プラットフォーム**、高単価コーチング業界の世界標準、**ゲーミフィケーション設計の教科書**

---

## 1. サービス概要

| 項目 | 内容 |
|---|---|
| 概要 | コミュニティ+コース+イベントを統合した「all-in-one」プラットフォーム。Sam Ovens（高単価コンサル業界の大物）が創業 |
| 主な対象 | コンテンツクリエイター・コーチ・コンサル・オンラインスクール運営者 |
| メンバーから見ると | 「Skool 上のコミュニティ」に参加して、課題・学習・交流をする場所 |

## 2. メンバー視点の画面構成

Skool 上の各コミュニティが持つ標準機能:

| 機能 | 説明 |
|---|---|
| **Community（投稿）** | Reddit風のスレッド型ディスカッション、投稿・コメント・返信 |
| **Classroom（コース）** | マルチモジュール式コース、動画・PDF・リンクを内包 |
| **Calendar（カレンダー）** | イベント・ライブ告知 |
| **Members（メンバー一覧）** | 同じコミュニティのメンバーをブラウズ |
| **Leaderboard** | ポイント獲得ランキング、7日・30日・全期間の3種 |
| **About（紹介）** | コミュニティの紹介ページ |
| **Map（地図）** | メンバーの所在地マップ（任意） |

→ **30分で1コミュニティ立ち上げ可**を謳う、シンプル統合UI

## 3. ゲーミフィケーションの仕組み ⭐⭐⭐ 筋肉塾の最大の学び

### ポイント・レベル・リーダーボード

**ポイント獲得**:
- 自分の投稿・コメントに **いいね 1件 = 1ポイント**
- 他者から評価されることで点数加算

**レベルアップ**:
- ポイントが一定値に達すると次のレベル解放
- **レベル解放で「特定のコース・コンテンツ」が unlock される**
- 「レベル◯になったら ◯◯コースが見れる」 = **段階解放のゲーミフィケーション**

**リーダーボード**:
- 7日 / 30日 / 全期間 の3タブ
- 上位者は「貢献度高い人」として可視化
- アバター横に「Lv X」表示で、誰がアクティブかが一目瞭然

### Gems（ジェム / 仮想通貨的なもの）
- 特別なリアクション・装飾の解放に使用（コミュニティ運営者がカスタマイズ可）

## 4. 学習継続を促す仕組み（コース機能）

| 要素 | 説明 |
|---|---|
| **コース構造** | コース → モジュール → レッスンの3階層、動画・PDF・リンクを混在 |
| **進捗マーク** | 各レッスンを「完了」マーク可 |
| **コメント** | 各レッスン下にコメント欄あり、コミュニティ全体と繋がる |
| **段階解放** | レベル制度と連動、特定レベル到達で次のコース解放 |
| **モバイルアプリ** | iOS/Android アプリで通知含む完全体験 |

---

## 5. 試験・テスト機能

- 明示的な「合否+正答率」型のテスト機能は **コア機能ではない**
- 代わりに「コース完了マーク」「ポイント獲得」がインセンティブ
- 必要ならコース内にクイズ動画埋込で代替

---

## 6. コミュニティ機能 ⭐

### Community タブ
- **Reddit ライクなスレッド型**
- 投稿（テキスト+画像）→ コメント → 返信のツリー構造
- カテゴリ別タブ（運営者が設定）
- 「Pin」で重要投稿を固定可
- 「@メンション」で他メンバー呼び出し

### Calendar タブ
- ライブイベント・Zoom URL付き
- メンバーは出欠登録可

## 8. 筋肉塾への適用ポイント ⭐⭐⭐

Skool のゲーミフィケーション設計は **筋肉塾の「完工率向上」に直結**する最も重要な参考。

### ✅ 採用すべき（強推奨）
1. **ポイント・レベル制度**
   - 筋肉塾: 「できた！を押す」「コメント投稿」「試験合格」で +ポイント
   - レベル達成で次の章のコンテンツ解放（M-3 段階公開の拡張）
2. **リーダーボード（7日・30日・全期間）**
   - 筋肉塾: 同期内 / 全期間の活動量ランキング
   - **要注意**: 過度なランキングは下位者の離脱誘発 → 「TOP10だけ表示」程度に抑制
3. **アバター横にレベル表示**
   - 「あの人すごい」「自分もあれくらい」の社会的圧（健全な範囲で）
4. **Pin機能 = のり氏の重要回答を固定**
   - 既にWFに反映済（コメント欄の📌）
5. **コース構造**: コース→モジュール→レッスンの3階層
   - 筋肉塾: 既に database で `courses → chapters → lessons` の3階層あり、一致

### ⚠️ 要調整
- **Gems機能**: 仮想通貨的すぎる、筋肉塾には合わなさそう
- **メンバーマップ**: 個人情報リスク、慎重に判断
- **Calendar機能**: 筋肉塾は LIVE 講義あるなら必要、なければ不要

### ❌ 真似しないほうがいい
- **過度なゲーミフィケーション**: 「ボディメイク」というテーマに対して「点数ゲーム化」しすぎると軽薄になる
- **Reddit風のスレッド型 Community タブ**: 筋肉塾は「レッスン下のコメント」が中心、独立スレッドは情報過多

## 9. アンチパターン

| 由来 | やらないこと |
|---|---|
| Skool | リーダーボードを **全期間 1位だけ表示**にして、頑張ってる人を放置 → 30日・7日も併設 |
| Skool | ポイントが「いいね数」依存 → 投稿しない学習者は不利。**「できた！」「視聴完了」もポイント源**に |
| Skool | 全体ランキングで下位者の存在を可視化 → TOP10など上位のみ可視化 |
| Skool | レベル解放を強制すると「ポイント稼ぎ目的」の投稿乱立 → 解放条件は控えめに設定 |
