## 🎯 結論（先出し・絶対に押さえる5点）

### 1. 新サイトの真の役割
**「生徒が正しく学び、継続的に学び、結果を出すための装置」を作ること**

社長の言葉:
> 新機能のKPIは売上ではなく、**生徒の実施完工率**です。
> 入会者の満足度や結果にコミットすることで、評判がよくなる。

→ 単なる技術移行や売上拡大装置ではなく、**学習体験の質を最大化する道具**。

### 2. **KPI: 主軸 = 生徒の実施完工率、副軸 = 売上向上（間接効果）**

| 軸 | 内容 | 出典 |
|---|---|---|
| 主軸 | **生徒の実施完工率**（学習継続・完工）| 社長 |
| 副軸 | 売上向上（広告材料・満足度経由の入会率UP・LTV増）| きよむさん |
| 別軸 | 売上回復本体（マーケ施策で別途進行中）| 社長 |

**主軸を追求すれば、副軸が結果としてついてくる構造**で設計する。

### 3. **新サイトでは決済しない** 🔥
社長: 「新サイトでは決済はしない」

→ 決済は **Stripe 決済リンク等で新サイト外で完結**、新サイトは **Stripe Webhook を受信して受講生 ID を自動発行**。

→ **大幅な工数削減・セキュリティリスク低減**:
- Stripe API 実装不要
- PCI DSS 対応不要
- クレカ情報を取り扱わない
- 返金処理は Stripe Dashboard

### 4. クリティカルなUX（社長言葉）
> ゴールは入会者が**正しく学べること、継続的に学びやすくすること、管理側で進捗が管理できること**。
> 「初回ログイン → コンテンツアクセス」で**新サイトの効果が試される**。

→ 新サイトの責任は「**学習開始から完工までの体験**」を最大化すること。

### 5. やるべき優先順位の軸
**「生徒の学習継続・完工に効くか」**で判断する。
副次的に「副軸（売上）に効く施策」も尊重するが、主軸を優先。

### 6. 最大目的の明文化（きよむさん）
> 使いやすく・他と連携・見やすく・項目を増やす → **満足度を上げる**

→ MVP 設計の評価軸として、以下4つを意識:
- **使いやすさ**（UX、操作の摩擦削減）
- **連携**（trainercloud / noriAI / LINE / Stripe 等との接続）
- **見やすさ**（情報設計、スマホファースト）
- **項目の充実**（足りない機能の追加）

これらが満たされれば、結果として**満足度が上がり、実施完工率が上がり、評判が広がり、副軸（売上）にも効く**構造。

## 💡 「実施完工率」の私の解釈案

社長言葉「実施完工率」の定義（要・確認）:

**候補**:
- A. 全レッスン視聴完了率（=「学習済み」ボタンを押した割合）
- B. 試験合格率（=試験全合格の割合）
