全体構成

  • PC画面想定 (管理画面はスマホ非対応で OK の合意あり、スマホでも一覧チェック程度は可能)
  • ヘッダー: ブランド + ログイン中ユーザー
  • 検索バー: 受講生名で絞り込み
  • フィルタタブ: 全て / 未返答 / 返答済 (シンプル 3 分類)
  • リスト: 2 セクション (未返答 古い順 → 返答済)

シーン別の使い方

  • シーン 1: 月初の集中作業 — 「未返答」タブ → 古い順に上から処理 → 行タップで個別作業画面 (3b) へ
  • シーン 2: 隙間時間の確認 — 件数バッジで「未返答 14 件」を一目把握

デザインルール

  • 未返答: 黄 #b8860b (受講生 UI と同じ "添削色")
  • 返答済: ティール緑 + 50% 透明度 (完了感)
  • 角丸 8-12px: 受講生 UI (14px) より少しシャープ (情報密度高い)
  • フォント: 数値・日付 = Inter、テキスト = Noto Sans JP

機能仕様

  • 並び順: 古い順 (経過日数の大きい順)
  • 件数バッジ: 各タブ右に未読数表示
  • クリック: 行タップで画面 3b へ遷移
  • 判定: シンプルに 提出 → 返信したか / してないか だけ

2026-05-24 変更点

  • 「危険信号」概念を 削除 (きよむさん判断: 添削が来て返したかが分かればいい)
  • 3 セクション → 2 セクション に簡略化 (未返答 + 返答済)
  • 赤色 #d32f2f は CSS に残るが使用箇所なし (Phase 2 で「7日経過のソフト警告」復活する可能性は残す)
