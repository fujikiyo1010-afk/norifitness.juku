# 22 エラー・空状態・ローディングのUIパターン

**作成日**: 2026-05-20
**目的**: WF v2.1 で触れていない「**例外状態のUI**」を体系化。データがない時・通信失敗時・読み込み中の体験設計。
**重要度**: 通常状態が完璧でも、**例外状態が雑だとサイト全体が低品質に見える**。MVP でも必ず必要。

---

## 🎯 設計原則

| 原則 | 説明 |
|---|---|
| 1. **離脱させない** | エラーや空状態で「もういいや」と諦めさせない |
| 2. **温度を保つ** | 機械的でなく、のり氏らしい温かいトーンで |
| 3. **次の行動を示す** | 「何をしたらいいか」が必ず明確 |
| 4. **責めない** | ユーザーのせいにしない（「入力ミス」より「もう一度確認」） |
| 5. **覚悟させない** | 重大に見せすぎない（軽微な事象を大袈裟に） |

---

## 1. 🌱 空状態（Empty State）

「まだデータがない」時の画面。**MVP初期はほぼ全ユーザーが遭遇**するため、最も重要。

### 1-1. パターン: アイコン + ひとこと + CTA

```
┌─────────────────────────────────────────┐
│                                         │
│              📝                         │  ← 大きめアイコン
│                                         │
│      まだ振り返りがありません            │  ← 状況の説明（責めない）
│                                         │
│     レッスンを見終わったら、              │  ← 次に何をすべきか
│     学んだことを3行で書いてみましょう。   │
│                                         │
│       [ レッスンを見る → ]                │  ← 次の行動への CTA
│                                         │
└─────────────────────────────────────────┘
```

### 1-2. 画面別の空状態カタログ

#### `/profile/reviews`（振り返り一覧、振り返りゼロ件）
```
📝
まだ振り返りがありません

レッスンを見終わったら、学んだことを
3行で書いてみましょう。
─────────────
[ レッスンを見る → ]
```

#### `/profile/bookmarks`（ブックマーク一覧、ゼロ件）
```
🔖
まだブックマークがありません

動画の気になる瞬間で
🔖 ボタンを押すと、ここに保存されます。
─────────────
[ レッスンを見る → ]
```

#### `/goal-sheet`（目標シート、まだ書いてない）
```
🎯
まだ目標が記入されていません

ボディメイクの旅は、
" 自分の目標 " から始まります。
─────────────
[ ✏️ 目標を書く ]
```

#### コミュニティ（同期内に投稿ゼロ）
```
💬
まだ投稿がありません

最初の一言、書いてみませんか?
ちょっとした気づきでも、誰かの励みになります。
─────────────
[ + 投稿する ]
```

#### レッスン下のコメント欄（コメントゼロ）
```
コメントはまだありません。
このレッスンへの質問・気づきを書いてみよう。
[ + コメントする ]
```

#### 通知（未読なし）
```
🔔
新しいお知らせはありません

最新の情報はここに届きます。
```

#### 試験結果履歴（未受験）
```
📝
まだ試験を受けていません

各章末に試験があります。
学習の定着を確認してみましょう。
```

### 1-3. 設計ガイド

| 要素 | 推奨 |
|---|---|
| アイコン | 80x80px 程度、`accent-300` でフラットに |
| 見出し | 16px bold、`ink-700` |
| 説明文 | 14px、`ink-500`、2行以内 |
| CTA ボタン | `brand-700` か `accent-600`、配置は中央 |

## 2. ⏳ ローディング（Loading）

「データを読み込み中」の表示。**0.5秒以上かかる時は必ず**何か表示する。

### 2-1. パターン分類

| パターン | 使用場面 |
|---|---|
| **スピナー（円形回転）** | 短時間（〜2秒）の操作、ボタン押下後の処理 |
| **スケルトン（骨組み）** | ページ全体の初期読み込み |
| **進捗バー** | 時間が予測できる長時間処理（動画アップロード等） |
| **プレースホルダー** | 画像読み込み中 |

### 2-2. スケルトンUI（推奨）

通常画面の骨組みを灰色で表示して「もうすぐ来る」感を演出。

#### ホーム画面のスケルトン例

```
┌─────────────────────────────────────────┐
│  [灰色ブロック]                          │
│  ─────────                              │
│                                         │
│  ┌─────────────────────────┐           │
│  │ ▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒        │           │
│  │ ▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒    │           │
│  │ ┌──────┐               │           │
│  │ │▒▒▒▒▒│               │           │
│  │ └──────┘               │           │
│  │ ▒▒▒▒▒▒▒▒▒▒              │           │
│  └─────────────────────────┘           │
│                                         │
│  ┌─────────────────────────┐           │
│  │ ▒▒▒▒▒▒▒▒                │           │
│  │ ████████░░░░░░░░ (animated) │       │
│  └─────────────────────────┘           │
└─────────────────────────────────────────┘
```

#### スタイル
```css
.skeleton {
  background: linear-gradient(
    90deg,
    var(--color-ink-100) 0%,
    var(--color-ink-50) 50%,
    var(--color-ink-100) 100%
  );
  background-size: 200% 100%;
  animation: skeleton-pulse 1.5s ease-in-out infinite;
  border-radius: 8px;
}

@keyframes skeleton-pulse {
  0% { background-position: 200% 0; }
  100% { background-position: -200% 0; }
}
```

### 2-3. スピナー（円形回転）

短時間操作用。ボタン押下直後など。

```
[ 保存中... ⟳ ]   ← ボタン内に表示
```

```css
.spinner {
  width: 16px;
  height: 16px;
  border: 2px solid var(--color-brand-100);
  border-top-color: var(--color-brand-700);
  border-radius: 50%;
  animation: spin 0.8s linear infinite;
}
@keyframes spin { to { transform: rotate(360deg); } }
```

### 2-4. ローディングテキストの選び方

| 場面 | コピー例 |
|---|---|
| ページ初期 | 「読み込み中...」 |
| データ送信 | 「保存しています...」 |
| 動画読み込み | 「動画を準備しています...」 |
| 検索 | 「検索しています...」 |
| 試験採点 | 「答え合わせ中...」 |
| ❌ NG | 「Loading...」「処理中」「Please wait」 |

### 2-5. タイムアウト対策

5秒以上かかったら、補足表示:
```
読み込み中...
（少し時間がかかっています）
```

10秒以上かかったら、リトライCTA:
```
読み込みに時間がかかっています。
[ もう一度試す ]
```

## 3. ⚠️ エラー状態（Error State）

何かが失敗した時。**最も難しい設計領域**。

### 3-1. エラーの種類別 UI

#### A. 通信エラー（ネットワーク切断）

```
┌─────────────────────────────────────────┐
│              📡                          │
│                                         │
│   うまく通信できませんでした              │  ← 責めない言い方
│                                         │
│   電波の状態を確認して、                  │  ← 解決策ヒント
│   もう一度試してみてください。            │
│                                         │
│       [ もう一度試す ]                    │  ← 行動CTA
│                                         │
└─────────────────────────────────────────┘
```

#### B. 入力エラー（バリデーション）

インライン表示が基本。

```
メールアドレス
┌──────────────────────────┐
│ kiyomu@                  │
└──────────────────────────┘
⚠️ メールアドレスの形式を確認してください
```

#### C. アクセス権エラー（404 / 403）

```
┌─────────────────────────────────────────┐
│              🔍                          │
│                                         │
│   ページが見つかりません                  │
│                                         │
│   このページは移動したか、                │
│   削除された可能性があります。            │
│                                         │
│   [ ホームへ戻る ]                       │
│                                         │
│   ※ お困りの場合は LINE で               │
│     お問い合わせください                  │
│                                         │
└─────────────────────────────────────────┘
```

#### D. サーバーエラー（500）

```
┌─────────────────────────────────────────┐
│              ⚠️                          │
│                                         │
│   一時的な問題が発生しています            │
│                                         │
│   しばらく時間をおいてから、              │
│   もう一度試してみてください。            │
│                                         │
│   [ もう一度試す ]   [ ホームへ ]        │
│                                         │
│   ※ 改善しない場合は LINE で             │
│     お問い合わせください                  │
│                                         │
└─────────────────────────────────────────┘
```

#### E. 重大エラー（決済失敗・データ消失リスク）

```
┌─────────────────────────────────────────┐
│              ⚠️                          │
│                                         │
│   保存に失敗しました                      │
│                                         │
│   入力された内容は残っています。          │
│   「もう一度保存する」を押してください。  │
│                                         │
│   [ もう一度保存する ]                    │
│                                         │
│   ※ それでも失敗する場合は、             │
│     入力内容をコピーして                  │
│     LINE でお知らせください              │
│                                         │
└─────────────────────────────────────────┘
```

### 3-2. エラーメッセージのコピールール

| ❌ NG | ✅ OK |
|---|---|
| Error 500 Internal Server Error | 一時的な問題が発生しています |
| Invalid input | メールアドレスの形式を確認してください |
| Failed to save | 保存に失敗しました。もう一度試してみてください |
| Network error | うまく通信できませんでした |
| 入力が間違っています | 入力内容を確認してみましょう |
| 不正なアクセス | このページを開けません |

### 3-3. エラー時の必須要素

1. **何が起きたか**（過剰でない説明）
2. **どうしたらいいか**（リトライ or 別のアクション）
3. **困った時の出口**（LINE 問い合わせ等）

## 4. ⏸ オフライン状態

```
┌─────────────────────────────────────────┐
│  [画面上部に常駐バナー]                  │
│  ⚠️ ネットワークに接続されていません      │
└─────────────────────────────────────────┘

通常画面は表示し続ける（読めるところまで）。
オフラインで保存可能なものはローカルキャッシュ。
復旧時に自動再同期。
```

PWA 化時に Service Worker で対応。

## 5. ✅ 成功状態（Success Toast）

操作が成功した時の控えめな通知。

```
┌──────────────────────────┐
│  ✓ 保存しました           │  ← 画面下に2秒表示後消える
└──────────────────────────┘
```

### 場面別

| 操作 | コピー |
|---|---|
| 振り返り保存 | ✓ 保存しました |
| プロフィール更新 | ✓ プロフィールを更新しました |
| コメント投稿 | ✓ コメントを投稿しました |
| 目標シート保存 | ✓ 目標を更新しました |
| 試験合格 | 🎉 試験に合格しました（→ 結果画面へ自動遷移） |
| ブックマーク追加 | 🔖 ブックマークしました |
| バッジ獲得 | 🎓 「章1マスター」を獲得しました |

### スタイル
- 画面下中央、`brand-700` 背景、白文字
- 2秒で自動消滅、または ✕ で即消去可
- 重要度低（できた！等）は控えめに、高（試験合格等）は派手に

## 6. 🌐 状態遷移のシナリオ集

実装時のチェックリストとして。

### A. 振り返り保存のフロー

```
[未保存状態] [💾 保存] ボタン押下
  ↓
[ローディング] ボタン内スピナー、ボタン無効化
  ↓ 成功
[成功] ✓ トースト「保存しました」、ボタン復活
  ↓ 失敗
[エラー] インライン or トースト「保存に失敗しました。もう一度試してみてください」
        ボタン復活、入力内容は保持
```

### B. レッスン詳細ページの初期ロード

```
[URL アクセス]
  ↓
[スケルトン] ヘッダー含む全体の骨組み表示
  ↓ 0.5秒〜2秒
[コンテンツ] 動画・コメント・周辺情報がフェードイン
  ↓ もし失敗
[エラー]「ページの読み込みに失敗しました [再読み込み]」
```

### C. コメント投稿

```
[未投稿] [+ コメントする]
  ↓
[入力モーダル] テキスト入力 + 画像添付
  ↓ [送信]
[ローディング] ボタンスピナー
  ↓ 成功
[成功] モーダル閉じる、トースト「投稿しました」、リストに新コメントフェードイン
  ↓ 失敗
[エラー] モーダルそのまま、エラーメッセージ表示、入力内容保持
```

### D. 動画ブックマーク追加

```
[動画再生中] [🔖 この瞬間をスクラップ]
  ↓ 即時
[フィードバック] ボタンが一瞬光る、トースト「🔖 ブックマークしました」
  → 失敗時は静かにロールバック（ボタン状態元に戻す）
```

## 7. ⏱ パフォーマンスとUX

### 体感速度の閾値

| 待ち時間 | UX対応 |
|---|---|
| 〜100ms | 即時、何もしない |
| 100ms〜1秒 | スピナー or 状態変化のみ |
| 1秒〜10秒 | スケルトン or 明確なローディング表示 |
| 10秒〜 | 進捗バー + キャンセル可能 |
| 60秒〜 | バックグラウンド処理化、結果は通知で |

### 楽観的更新（Optimistic UI）

操作が高頻度の場面では「先に成功させて、失敗したら戻す」設計:
- ✅ できた！ボタン（押した瞬間に色変化）
- ✅ コメント投稿（送信直後にリストに表示）
- ✅ いいね（押した瞬間にカウントアップ）
- ❌ 重要な保存（目標シート編集等）→ 失敗の影響大きいので普通の loading で

## 8. アクセシビリティ・スクリーンリーダー対応

### ARIA属性

| 状態 | ARIA |
|---|---|
| ローディング中 | `aria-busy="true"` |
| エラー表示 | `role="alert"`, `aria-live="assertive"` |
| 成功トースト | `role="status"`, `aria-live="polite"` |
| 空状態 | `role="region"`, `aria-label="◯◯がありません"` |

### Focus 管理
- モーダル開いたら最初のフォーカス可能要素へ
- エラー時はエラーメッセージへフォーカス移動
- 成功後は次のCTAへフォーカス

## 9. 実装時のチェックリスト

メイン実装時に確認すべき項目:

### 全画面共通
- [ ] スケルトンUI を実装したか
- [ ] エラー時のリトライ動線があるか
- [ ] 空状態のUI が温かいトーンか
- [ ] トーストが控えめに2秒で消えるか
- [ ] 通信失敗時に入力内容が保持されるか

### 学習関連画面
- [ ] レッスン読み込み失敗時の対応
- [ ] 動画再生エラー時の対応（Vimeo 失敗）
- [ ] コメント投稿失敗時のリトライ
- [ ] できた！押下の楽観的更新

### 認証関連
- [ ] 招待トークン期限切れの専用画面
- [ ] パスワード再設定リンク期限切れの専用画面
