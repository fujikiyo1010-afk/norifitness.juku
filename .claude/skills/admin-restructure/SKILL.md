# 管理画面 再構成の進め方（憲法）

トレクラ廃止 → 筋肉塾に内製統合。管理画面を「デイリー添削・伴走の司令塔」として再構成する。
**トレクラを参考にするが構造から変え、筋肉塾固有資産（献立/原本メニュー/レッスン/振り返り/実践宣言/目標シート/月次/カルテ）を骨格にする。**

## 1. まず"正典3ファイル"を読む（＝仕様。色/チップ/角丸/文言まで再現）
1. 提案書（背景〜仕様〜データ要件〜段階導入 全部入り）
   `~/Desktop/norifitness/_Cursor提案_デイリー添削と管理画面再構成.md`
2. デイリー添削モック（/admin/daily 想定）
   `~/Desktop/norifitness/08_guide/提案_管理_デイリー添削_A3_今日重視.html`
3. ユーザーハブ「記録」タブ
   `~/Desktop/norifitness/08_guide/提案_管理_ユーザーハブ_記録タブ.html`

モックは"参考"ではなく**仕様**。レイアウト・配色・チップ・角丸・文言（「今のからだ」「あと10.8kg」等の受講生と同じ言葉づかい）まで再現する。追加はモック側に足していく前提。

## 2. 着手前に必ず言語化する（コードに触る前）
順に出す：**認識**（モックのどこの話か）→ **判断**（今できる/データ待ちか。既存テーブルで作れる範囲を明示）→ **プラン**（P0〜P4のどれか）→ **質問**（未決事項を選択肢で）。
- 決定を急がない。実装は合意後。
- 「これはP0/P1/…のどれ？」を毎回位置づける。

## 3. きよむ確定事項（勝手に変えない）
- **サイドナビ4グループ＝採用 / ホーム「今日の流れ」レール＝不採用（外す）**。P0はnav4グループ化のみ。
- **進め方＝B**：デイリー添削"本体"は今作らず、②食事・⑤実施記録・生活記録が揃ってから作る（空の器を避け完成形で出す＝完成優先）。順序＝受講生側入力→データ発生→デイリー添削本体。
- **本番反映は合図まで保留**。feat→main 丸ごとマージ＝体組成が本番公開になるため未実施（切り出しは deploy-isolated スキル）。
- 視聴進捗は**完了のみ**（実視聴時間は取らない）。
- 第1層＝「今のからだ＋今日の3つ（トレ・食事・学習）」4カード＝受講生ホーム確定7/7のミラー。**生理変動±0.3は赤にしない**。献立は"参考"、ものさしは**のり監修の目標PFCのみ**。

## 4. 表示統一ルール（未完成の管理画面も今後これに揃える＝共通コンポーネント化）
- サマリー行（sumrow）＝ラベル小＋数値太字 mono。
- テーブル行スタイル／チップ（`add`・`skip`・`chg` 等、モックの配色）。
- カード角丸 11px／ティール `#00897b`（dark `#00695c`）／薄い区切り線でタブ列・サマリー列の詰まりを分ける。
- 管理用語（最新体重・記録日数）を使わず、受講生と同じ言葉。

## 5. 段階導入（提案書§5。DDLは各着手時に必ず先出し）
| Phase | 内容 | 依存 |
|---|---|---|
| P0 | サイドナビ4グループ化（今日の流れレールは入れない） | なし |
| P1 | デイリー添削 器＋キュー＋4カード（からだ/学習/アラートは既存データ）＋下部固定FB | `daily_feedbacks` |
| P2 | 生活記録（受講生入力＋4点目＋睡眠グラフ） | `daily_conditions`＋受講生UI |
| P3 | 食事統合（食事カード・目標PFCゲージ・参考献立・摂取カロリーグラフ） | `meal_logs`＋目標PFC＋メニューJSON |
| P4 | 筋トレ 原本vs実績 | `user_workout_logs` |

新規4テーブル＝`meal_logs` / `user_workout_logs` / `daily_conditions` / `daily_feedbacks`。**DDLは実装前に提示→承認→適用**（dev=yciqbigyzfqmmjdcnqfk / prod=fqfsgkzyotvpcxmszkax、SQL依頼はクリック可能なSupabase URLを添える）。

## 6. 未決（各フェーズ着手時に選択肢を出して詰める・今答え不要）

- 既存の動線・本番運用を壊さない。変更は目的範囲限定、毎タスク完了で型/lint→動線テスト→回帰確認→正直報告。

- 月次添削（動画・FIFO受信箱）は現行のまま。デイリー添削（日次テキスト）とは別物として共存。
- 管理画面はデスクトップPC専用（スマホモック不要）。UX細部はClaude判断でOK。
